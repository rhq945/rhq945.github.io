HTTP概念

特点

  • 基于TCP/IP的高级协议
  • 默认端口号：80
  • 基于请求/响应状态的模型的：一次请求对应一次相应（因为交互元素太多，使用复用连接）
  • 无状态的：每次请求间相互独立，并不能交互数据

传输协议

定义了客户端和服务器端通信时的发送数据的格式

请求消息数据格式

  1. 请求行

    GET:

    • 请求参数在请求行中，连接在网址URL后面
    • 请求的URL长度有限
    • 相对不安全，请求参数可见

    POST:

    • 请求参数在请求体中，不在URL中
    • 请求的URL长度无限制

    | | 请求方式 | 请求url | 请求协议/版本 |
    | —— | ———— | ———— | ——————- |
    | 例子 | GET/POST | /Javaweb/demo?name=Eric | HTTP/1.1 |

  2. 请求头：客户端告知服务器

    • 形式：

      请求头名称: 请求头值

    • 常见请求头：

      1. User_agent：浏览器告诉服务器，访问服务器所使用的浏览器版本信息；

        用来解决浏览器的兼容问题；

      2. Referer：告诉服务器，当前请求从哪里来；

        作用：

        1. 防止链接被盗（盗链）；

        2. 方便统计工作；

          统计进入某个界面的途径有哪些以及不同途径的流量；

      3. Connection

  3. 请求空行

    做分割作用，分离请求头和请求体；

  4. 请求体（正文）

    封装POST请求消息的请求参数；

Request

Request对象和Response对象的原理

  1. Request对象和Response对象都是由服务器创建的，我们只是去使用他们；
  2. 程序员使用Request对象来获取请求消息，Response对象用来设置响应消息

Request对象的继承体系结构

Request对象的功能

  1. 获取请求消息数据

    • 获取 请求行 数据

      例：GET /Javaweb/demo?name=Eric HTTP/1.1

      方法：1. 获取请求方式：例如获取GET

      ​ String getMethod()

      1. ⭐获取虚拟目录： /Javaweb

        String getContextPath()

      2. 获取Servlet路径也叫获取资源路径： /demo

        String getServletPath()

      3. 获取GET方式的请求参数：name=Eric

        String getQueryString()

      4. ⭐获取请求URI： /Javaweb/demo

        String getRequestURI()

        ⭐获取请求URL： http://localhost/Javaweb/demo

        StringBuffer getRequestURL()

      5. 获取协议及版本： HTTP/1.1

        String getProtocol()

      6. 获取客户机IP地址

        String getRemoteAddr()

    • 获取 请求头 数据

      方法：1. String getHeader(String name)：通过请求头的名称获取请求头的值

      1. Enumration getHeaderNames()：迭代器，可获取所有的请求头名称

    • 获取 请求体 数据

      POST请求消息才有请求体

      步骤：

      1. 获取流对象；字节流/字符流

        BufferedReader getReader()：获取字符输入流，只能操作字符数据；

        ServletInputStream getImputStream()：获取字节输入流，可以操作所有类型数据；

      2. 从流对象中拿数据；

  2. 其他

    1. 获取请求参数的通用方法：(兼容GET/POST)

      String getParameter(String name):根据参数名称获取参数值；

      String getParameterValues(String name):根据参数名称获取参数值数组；

      Enumeration<String> getParameterNames(): 获取所有请求的参数名称的枚举；

      /如果请求的参数是数组，那么只枚举第一个元素

      Map<String,String[]> getParameterMap():获取所有参数的Map集合；

      • doPost的中文乱码问题解决

        为什么doGet没有：doGet的参数在URL中，而POST方法参数在流中；

        修正：在获取参数前，设置request的编码

    2. 请求转发

      • 是一种在服务器内部的资源跳转方式；

      • 方法

        通过request对象获取一个请求转发器的对象dispatcher,request.getRequestDispatcher(s:"要转到的路径")；

        再使用dispatcher的forward方法来进行转发。

      • 特点

        浏览器地址栏路径不变，还是转发之前的；

        只能转发到当前服务器的资源内；

        转发是一次请求；

    3. 共享数据

      • 请求转发的时候也要进行数据通信

      • 域对象：有作用范围的对象，可以在范围内共享数据；

        request域：就代表一次请求的范围，所以可以用于请求转发中来共享数据；

      • 方法：

        1. void setAttribute(String name,Object obj)：存储数据，键值对方式
        2. Object setAttribute(String name)：通过键获取值
        3. void removeAttribute(String name)：通过键移除键值对

    4. 获取ServletContext对象
