HTTP:响应信息

响应消息：服务器端发送给客户端的数据；

数据格式

响应行

  1. 组成：

    协议/版本 响应状态码 状态码描述

    HTTP/1.1 200 OK

  2. 响应状态码

    服务器去告诉浏览器本次请求和响应的状态

    | 分类 | 含义 |
    | —— | —————————————————————————————— |
    | 1xx | 服务器接收客户端消息但没接收完，等待一段时间后，发送1xx状态码 |
    | 2xx | 响应成功！ |
    | 3xx | 重定向 304：服务器怎么知道客户端有缓存 |
    | 4xx | 客户端错误 |
    | 5xx | 服务器端错误 |

响应头

  1. 格式：键值对

    头名称：值

  2. 常见响应头

    1. Content-Type：服务器告诉客户端本次响应体数据格式及编码格式

    2. Content-disposition：服务器告诉客户端以什么格式打开响应体数据

      值：

      • in-line：默认值，在当前页面内打开；
      • attachment；filename=xxx；以附件形式（弹窗）打开响应体，文件下载；

响应体

页面展示的HTML的内容

  4. 重定向和转发特点：

    | | 转发 | 重定向 |
    | ————————————- | ————————————— | ——————————— |
    | 地址栏 | 不变 | 改变 |
    | 访问资源范围 | 只能访问当前服务器下的资源 | 可以访问其他服务器资源 |
    | 请求次数 | 一次请求 | 两次请求 |
    | 使用方法 | forward | redirect |
    | 能否使用request域共享数据 | 能 | 不能 |

  5. 路径写法（虚拟路径解答）：

    1. 路径分类

      • 绝对：可以确定唯一资源，在服务器中也可以省略协议和服务器ip，域名端口号；

        ​ 如： /servlet_03_response/respDemo1 http://localhost:8080/servlet_03_response/respDemo1

      • 相对：不可以确定唯一资源，找到当前资源和目标资源的相对位置关系。不以 ‘/‘ 开头，以 ‘./‘ 开头

        ​ 如：./index.html ,再往上一级用 ‘../‘ ；

    2. 绝对路径的写法规则

      判断要定义的路径是给谁用的；判断请求将来从哪里发出

      • 客户端浏览器：、

        、重定向（浏览器拿着目标资源去访问）

        需要加虚拟目录

      • 服务器：转发（服务器内部资源跳转），

        不需要加虚拟目录

    3. 动态获取虚拟目录！

      String contextPath = request.getContextPath();
