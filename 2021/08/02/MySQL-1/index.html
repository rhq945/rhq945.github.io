数据库相关操作

数据库的数据类型

注意

SQLyog中创建表时，数据类型后面跟着的长度是显示长度，和数据的占用内存无关！

数值

类型名 占用内存 用途
tinyint 1 byte
smallint 2 byte
mediumint 3 byte
int 4byte 常用的整数
bigint 8 byte
float 4 byte
double 8 byte 常用的浮点数
decimal 视情况而定 字符串形式的浮点数，金融中常用

字符串

类型名 占用内存 用途
char 0~255 bit
varchar 0~65535 bit 可变字符串，对应java的string，常用的变量
tinytext 255 byte 微型文本
text 65535 byte 文本串，保存较大文本

时间日期

java.util.Date

类型名 表示形式 用途
date YYYY-MM-DD 日期格式
time HH:mm:ss 时间格式
datetime YYYY-MM-DD HH:mm:ss 最常用的表示时间
timestamp 1970.1.1到现在的毫秒数 时间戳
year 年份表示

⭐数据库的字段属性

unsigned

  • 无符号整数；
  • 声明该列不能注册为负数；

zerofill

  • 对于不足长度的数字，使用0将其填充至指定长度；

自增

  • 自动在上一条记录的基础上+1；
  • 通常用来设计唯一的主键，并且必须为整数类型
  • 可以自定义主键自增的起始值和步长

数据表的类型(引擎)

MYISAM和INNODB

区别

MYISAM INNODB
事务支持 不支持 支持
数据行锁定 不支持 支持
外键约束 不支持 支持
全文索引 支持 不支持
表空间大小(占用内存) 较小 较大

常规使用操作

  • INNODB：安全性高，支持事务处理；
  • MYISAM：空间小，速度快；
