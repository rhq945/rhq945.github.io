会话

一次会话中包含多次请求与相应。

流程

  • 会话建立

    浏览器第一次给服务器资源发送请求；

  • 会话结束

    有一方断开；

功能：共享数据

HTTP协议是无状态的，每一次请求响应之间是相互独立的；

为了在一次会话范围内多次请求间进行共享数据，要用会话技术；

方式

  1. 客户端会话技术：Cookie

    把数据存到客户端；

  2. 服务器端会话技术：Session

    把数据存到客户端；

Cookie

概念

将数据保存到客户端的会话技术；

内部以map键值形式存储数据；

    new Cookie(String name,String value)

    实现原理

基于响应头和请求头实现：

  1. 客户端浏览器向服务器第一次发送请求，请求Cookie资源；
  2. 服务器向客户端发送Cookie，发送方式是响应头中加入set-Cookie:msg=hello(键值对)
  3. 在HTTP协议规定下，浏览器收到响应头中的set-Cookie时
    1. 将数据msg=hello保存到浏览器中;
    2. 在下一次请求中带上数据到服务器去，发送方式：请求头中加入cookie:msg=hello;

Cookie细节规则

  1. 一次可以发送多个Cookie

    创建多个Cookie对象即可，再多次使用response.addCookie()方法；

  2. Cookie的有效时间

    • 默认情况：浏览器关闭后，Cookie数据被销毁；

    • 持久化存储：存在文件中

      Cookie对象方法：setMaxAge(int seconds)

      1. seconds为正数：将Cookie数据写到硬盘文件中，持久化存储seconds秒长的时间，到时候不需关闭浏览器也自动销毁；
      2. seconds为0：直接删除Cookie信息；
      3. seconds为负数：默认值；

    • 如果有持久化存储的信息，那么客户端浏览器第一次向服务器发送请求时就已经携带了Cookie！

  3. Cookie存储中文

    Tomcat 8.0版本之后，支持中文！

  4. Cookie共享范围

    1. 同一个服务器的不同web项目：

      通过Cookie方法：setPath(String path)设置Cookie的获取范围；

      默认情况下。path设置为”/当前的虚拟目录“，所以只有在当前web项目中可以共享Cookie，其他项目不可；

      path路径的根目录是服务器（localhost）；

      所以想要服务器的web项目都可共享，则path设置为 “/“ 即可！

    2. 不同Tomcat服务器：

      调用Cookie的方法：setDomain(String path)

      设置一级域名相同，则多个服务器间可共享Cookie；

      如setDomain(".baidu.com"),则www.tieba.baidu.com和www.news.baidu.com间可共享Cookie

  5. Cookie特点&作用

    • 特点
      1. 存在客户端浏览器的数据安全度不高；
      2. 浏览器对于单个Cookie的大小有限制，对一个域名之下的Cookie数量也有限制；
    • 作用
      1. 存储少量的不太敏感的数据；
      2. 在不登陆的情况下完成服务器对客户端的身份识别；（不需要登陆也可以保存客户对本机应用的某些设置）

    Session

概念

在一次会话的多次请求之间共享数据，将数据保存在服务器端的对象中。Httpsession

快速入门

原理

一次会话中多次获取的session对象是同一个；

Session依赖于Cookie；

第一次获取Session时，没有Cookie，会在内存中创建一个新的Session对象，有一个唯一id=id号；

服务器做响应时会发送一个响应头，包含set-cookie : JSESSIONID=id号;

浏览器做请求时也发送一个请求头，包含cookie : JSESSIONID=id号;

这样就确保Session在一次会话中多次获取也是同一个！

细节实现

Session什么时候被销毁？

  1. 服务器关闭
  2. Session对象调用invalidate()方法；
  3. Session默认失效时间：30*60s

特点

  1. Session用于存储一次会话的多次请求的数据，存在服务器；
  2. 存储任意类型，任意大小的数据；
  3. 较Cookie更安全

JSP

概念

Java Server Pages：Java服务器端页面；

将其理解为一个特殊页面，即可以直接定义HTML标签，也可以直接写Java代码

原理

本质也是一个servlet

  1. 服务器解析请求消息，查找.jsp文件；
  2. 将.jsp转换为.java文件；
  3. 编译.java文件，生成.class字节码文件；
  4. 有字节码文件提供访问，给客户端做出响应；

脚本

Java代码定义的区域

  1. <%代码%>

    定义在service方法中；

  2. <%!代码%>

    定义在java类的成员位置，定义成员变量和方法；

  3. <%=代码%>

    输出代码，定义在service方法中；如<%= i %>,有定义i=5，则输出5；
